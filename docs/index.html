★ View on GitHub Browse the design docs
What it is

Designing the inverse-synthesis pipeline

A documentation-only repo defining a "horizontal" architecture of three orthogonal analysis experts plus a parameter-mapping stage — to be implemented inside audio-analysis-mcp. The pivot from an earlier vertical pipeline avoids cascading errors.

Amplitude expert
DSP envelope follower → ADSR
Modulation expert
RNN / CNN → LFO parameters
Tone-generation expert
CNN → engine + tone params (modulation-invariant)
Parameter mapping
Canonical parameter ontology + per-device translator
Engine classification
A softmax head on the tone expert — no separate detection phase
Training setup
SignalFlow renderer, Sobol / Latin-Hypercube sampling, shared baseline presets
